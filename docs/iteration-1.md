# Ітерація 1 - Статус та артефакти

## Що працює зараз

### Рівень Domain
- **Track** сутність: Незмінний об'єкт домену, що представляє один трек з Title, Artist, Duration та Genre
- **Playlist** сутність: Композитна сутність домену, яка може містити Tracks та інші Playlists
- **IPlaylistComponent** інтерфейс: Визначає контракт для Track та Playlist (паттерн Composite)

### Рівень Application
- **IPlaylistRepository** інтерфейс: Паттерн Repository для абстракції доступу до даних
- **InMemoryPlaylistRepository** реалізація: Зберігає плейлисти та треки в пам'яті під час сеансу
- **PlayerService** сервіс Application: Координує операції плейліста та треку

### Консольний UI
- Створити новий плейліст
- Додати треки до плейліста
- Відобразити структуру дерева плейліста з деталями треку
- Меню консолі навігації для основних операцій

## Створені артефакти

- `src/MusicPlayer.Domain/` - Сутності домену та інтерфейси
- `src/MusicPlayer.Application/` - Сервіси Application та інтерфейс Repository
- `src/MusicPlayer.Infrastructure/` - Реалізація in-memory Repository
- `src/MusicPlayer.Console/` - Точка входу консольного додатка та UI
- `tests/MusicPlayer.Tests/` - Модульні тести для логіки Domain та Application

## Сценарії для Лаб 35

### Сценарій 1: Створення плейліста з треками
```
Користувач створює "Мої улюблені" плейліст → Додає 3 треки → Система відображає структуру дерева
```

### Сценарій 2: Відображення ієрархії плейліста
```
Користувач переглядає плейліст → Система показує вкладену структуру з кількістю треків та загальною тривалістю
```

### Сценарій 3: Доступ до метаданих плейліста
```
Користувач отримує доступ до властивостей плейліста → Система повертає загальну тривалість, кількість треків, розподіл жанрів
```

## Класи підготовлені до розширення

- **IPlaylistRepository** - Замініти InMemoryPlaylistRepository на FileRepository, DatabaseRepository в майбутніх ітераціях
- **PlayerService** - Готовий до методів shuffle(), search(), filter() в Ітерації 2
- **Track** - Розширюваний для додаткових метаданих (альбом, рік, бітрейт) в Ітерації 3
- **Playlist** - Підготовлений для управління чергою та відстеження стану відтворення
