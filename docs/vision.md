# MusicPlayer_OOP - Бачення проекту

## Визначення проекту
MusicPlayer_OOP - це консольний додаток музичного плеєра на .NET 8, побудований на принципах Clean Architecture. Він керує треками, організує плейлисти за допомогою паттерну Composite і надає основну функціональність відтворення через архітектуру розширюваних сервісів.

## Цільові користувачі
- Любителі музики, які хочуть організувати та відтворювати локальні музичні колекції
- Розробники, які вивчають Clean Architecture та паттерни проектування в .NET
- Користувачі, які шукають легкий інструмент для управління музикою з командного рядка

## Основні сценарії

### Сценарій 1: Створення та управління плейлістом
Користувач створює новий плейліст, додає до нього треки та організує музику за жанром, настроєм або користувацькою колекцією.

### Сценарій 2: Відтворення треку
Користувач вибирає трек із плейліста або бібліотеки та відтворює його, переглядаючи метадані треку (назва, виконавець, тривалість).

### Сценарій 3: Перемішування та управління чергою
Користувач перемішує треки в плейліста, ставить в чергу кілька треків та навігує через історію відтворення.

## Нефункціональні вимоги

1. **Продуктивність**: Операції з плейлістом (додавання, видалення, перемішування) мають завершуватися протягом 100 мс для колекцій до 1000 треків.
2. **Підтримуваність**: Код має дотримуватися SOLID принципів з чітким розділенням занепокою на шарах Domain, Application та Infrastructure.
3. **Розширюваність**: Архітектура повинна підтримувати додавання нових постачальників сховища (файл-базовані, бази даних) та джерел відтворення без змін основної логіки домену.

## Область Ітерації 1

- **Мета**: Встановити модель домену та архітектуру вертикального зрізу
- **Результати**: 
  - Сутності домену (Track, Playlist) з паттерном Composite
  - Реалізація in-memory сховища
  - Сервіс PlayerService на рівні Application
  - Консольний UI для базових операцій (створення плейліста, додавання треку, дисплей)
- **Виключення**: 
  - Реальне відтворення аудіо
  - Постійне зберігання (база даних/файлова система)
  - Розширені алгоритми перемішування
  - Аутентифікація користувача
